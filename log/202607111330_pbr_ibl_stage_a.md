# PBR / split-sum IBL — Stage A: ops, builders, real-time viewer

## Goal

learnopengl.com/PBR (Cook-Torrance + split-sum IBL) を Dressi の記法で実装する
第一段。最終成果物は同一 forward グラフを共有する 3 サンプル
(①リアルタイムビュワー ②マテリアル最適化 ③Envmap 最適化)。本ログは
①まで (Stage 0 + A)。

## What was done

### Dependencies / assets (Stage 0)
- tinyexr v1.0.9 を FetchContent header-only パターンで追加 (zlib は
  asset_utils.cpp の stb 実装を再利用: `TINYEXR_USE_STB_ZLIB=1`)。
  OpenEXR は実ビルド+Imath 依存で重いため不採用 (PolyHaven の scanline
  EXR には tinyexr で十分)。
- FetchAssets.cmake: 既存 glTF-Sample-Models クローンブロックを拡張して
  DamagedHelmet もコピー (新しい取得機構は追加しない)。
- `LoadGltfScene` (examples/common/asset_utils): NORMAL/TANGENT 読み込み
  (欠落時は面積重み法線 / Lengyel 接線生成)、ノード変換の CPU 適用
  (DamagedHelmet は回転クォータニオン持ち)、マテリアルテクスチャの
  tinygltf デコード取り込み、**V 反転なし** (glTF は top-left 原点。
  既存 LoadGltfMesh の 1-v は OBJ 由来でそのまま)。
- `LoadImageExr` / `SrgbToLinear` / `DownsampleHalf` 追加。

### New ops (src/core/f_ops.cpp, codegen, core/ibl_math.h)
- `F::EquirectSample(map, dir)` — 方向→equirect uv→手動 bilinear
  (u wrap / v clamp)。ゼロ方向 (ラスタ背景) は 0。forward-only。
- `F::TextureBilinear(tex, uv, inv_uv)` — 手動 bilinear (clamp)。
  **テクスチャ微分可能**: backward `__gather_inv_uv_bilinear__` は既存
  `__gather_inv_uv__` の変種で、アンカー画素の ±3 窓の全寄与画素を
  forward のテント重みで**積算**する (nearest 版は1画素のみ)。
- `F::IrradianceConv(env, out)` — 全ソーステクセルの決定論 cos·sinθ 直和、
  格納は E(N)/π (定数環境 L0 → 出力 L0 がテストの錨)。
- `F::PrefilterEnv(env, out, rough, n)` — GGX importance sampling
  (Hammersley は手動 bitreverse で CPU/GPU bit 同一)、V=N=R、NoL 重み。
- `F::BrdfIntegrationLut(size, n)` — ゼロ入力 (ScreenCoord 前例)、
  VEC2 (A,B)、Smith-G k=r²/2。
- `F::AvgPool2x2` — 既存 Reduce2x2Sum×0.25 の1行ラッパー (微分可能)。
- 全 CPU カーネルは `core/ibl_math.h` の共有実装を呼ぶ (GLSL 側は
  codegen が emit する `dressi_*` ヘルパーが双子)。サンプラーは一切
  触らない (全 nearest のまま): HW bilinear は CPU パリティを壊し、
  clamp サンプラーは equirect シームを壊すため手動 4 タップ一択。

### Builders (include/dressi/ibl.h, src/core/ibl.cpp)
論文 Appendix 形状: `BuildIrradianceSample` (AvgPool→64×32→conv、既定
32×16)、`BuildPrefEnvironmentSample` (レベル別 Variable×5、128×64→8×4、
mip 不在による論文からの逸脱)、`BuildBrdfIntegrationMap` (256², n=512)。
ランタイム側 `SampleIrradiance` / `SamplePrefEnvironment` (LOD テント重み
blend = 隣接2レベル線形補間) / `SampleBrdfLut`。

### Shared forward graph + viewer example
- `examples/common/pbr_graph.{h,cpp}`: G-buffer (Rasterize×4: uvh/normal/
  tangent/wpos) → TextureBilinear×5 → TBN 法線マッピング → 点光
  Cook-Torrance (test_cpu_eval.cpp:209 の式 + 距離減衰) → IBL ambient
  (kD·irr·albedo + pref·(F0·A+B))·AO → emissive → ScreenCoord+カメラ基底
  uniform による環境背景 → Reinhard+gamma。`SafeNormalize` (0除算→0) が
  背景画素の NaN 汚染を防ぐ要。inv_uv は引数 (ビュワーは未アップロードの
  ダミー — forward は inv_uv を消費しないため合法)。
- `examples/pbr_shading/`: 自動オービット + VkViewer、毎フレーム同一
  リーフ集合 {clip, cam×4, light×2} を `execStepWithSendsAndRecvImgsStacked`
  で 1 submit。ヘッドレス時は全周 PNG。≥20 フレーム除外の median 表示。

### Codegen fix
special op の SamePixel 入力 (dir/uv) は Naive packing で slt に分類される
が generic load 対象外だった → `needs_generic_load` に
`__equirect_sample__`/`__sample_bilinear__` の第2オペランドを追加。

## Measurements (RTX PRO 6000, 512², DamagedHelmet 14.5k verts / 15.5k faces, env 512×256)

- 初回ビルド ~239 ms (vulkan-build 128 + upload 83; 2048²×5 テクスチャ)。
- **reactive cache が IBL 前計算をプルーニング: 12 stages (warmup) → 5 (定常)**。
  前計算 (irr 32×16 + pref 5 levels + LUT 256²) はウォームアップ中のみ実行。
- 定常 execStep+readback median **1.35–1.49 ms/frame = 670–740 FPS**
  (gpu-execute ~1.2 ms; silhouette 0.26 ms 比の増分は RASTER×4 と
  2048² マテリアル×5 + IBL 7 サンプル/px の融合 FRAG パス)。
- テスト: 既存 92 → **120 全緑** (CPU: 解析値/シーム連続性/定数環境の錨、
  TextureBilinear 中心差分勾配; GPU: 両 packing パリティ tol 1e-3、
  reactive rebuild 後の結果不変、bilinear 回復最適化)。

## Key decisions

- bilinear gather の安定学習率は nearest 版と別物 (Σw² 倍): 回復テストの
  発散 (NaN) はこれが原因で lr=15/500iter に調整。CLAUDE.md に記録。
- V 反転問題: normal G-buffer のリブ模様が正常に出たことで UV 正当性を確認
  (公式スクリーンショットも暗い緑黒のヘルメットであり見た目は正しい)。
